# Hackathon One-Line Questions

---

## 1. What is the Boot Process?

The sequence that starts the computer and loads the operating system.

---

## 2. What is POST?

A hardware self-test performed by BIOS/UEFI during startup.

---

## 3. What is BIOS/UEFI?

Firmware that initializes hardware and starts the boot process.

---

## 4. What is a Bootloader?

A program that loads the operating system kernel into memory.

---

## 5. What is the Kernel?

The core part of the operating system that manages hardware and resources.

---

## 6. What is a Process?

A program that is currently running.

---

## 7. What happens when a program starts?

The OS loads it into RAM and the CPU begins executing its instructions.

---

## 8. Why is RAM needed to run programs?

Because the CPU can execute instructions only from memory.

---

## 9. What is Virtual Memory?

A memory management technique that gives each process its own logical address space.

---

## 10. What happens when a file is opened?

The OS locates the file, loads its data into RAM, and gives it to the application.

---

## 11. What is an Interrupt?

A signal that immediately notifies the CPU about an event requiring attention.

---

## 12. How does keyboard input reach an application?

Keyboard → Scan Code → Interrupt → Driver → OS → Application.

---

## 13. How does mouse input work?

Mouse movement generates events that the driver converts into cursor actions.

---

## 14. What is Compilation?

The process of converting source code into machine code.

---

## 15. What are the four stages of compilation?

Preprocessing, Compilation, Assembly, and Linking.

---

## 16. What is an Executable?

A machine code file that the operating system can run.

---

## 17. What is the role of the Scheduler?

It decides which process gets CPU time.

---

## 18. Why are drivers important?

They enable the operating system to communicate with hardware.

---

## 19. Why are interrupts faster than polling?

Because the CPU responds only when an event occurs instead of continuously checking.

---

## 20. Explain the complete system workflow in one line.

Power On → Boot → OS Loads → Program Runs → File Access → User Input → CPU Executes → Program Ends.

---
